## 1. Product Overview

### 1.1 Product Name

**SiteGrade** — *Know your score. Fix what matters.*

### 1.2 Elevator Pitch

SiteGrade audits any website in under 60 seconds — covering performance, SEO, accessibility, security, UX, and best practices — then uses AI to tell you exactly what to fix.

### 1.3 Problem Statement

Web designers and agencies spend hours manually auditing client sites or rely on fragmented free tools. Existing tools (GTmetrix, PageSpeed Insights) give raw data but no actionable recommendations. No single tool combines all 6 audit categories with AI-powered fix instructions.

### 1.4 Solution

SiteGrade is the only website audit tool that combines Lighthouse performance data, WCAG accessibility checks, SEO analysis, security scanning, and UX evaluation — then uses AI (GPT-4o) to translate every finding into plain-language, prioritized fix recommendations with effort estimates and code examples.

### 1.5 Target Audience

| Segment | Pain Point | SiteGrade Solution |
|---------|-----------|-------------------|
| **Freelance web designers/devs** | Spend 2+ hours auditing client sites manually | Audit any site in 60 seconds |
| **Web design agencies** | Need professional branded reports for clients | White-label PDF reports with Agency plan |
| **In-house dev teams** | Catch regressions, prove value of performance work | Score history + trend charts |
| **SEO consultants** | Technical + SEO audit data fragmented across tools | Single report covering all categories |
| **Digital marketing agencies** | Need branded reports to justify retainers | White-label, shareable report links |

## 3. Feature Specification

### 3.1 Core Audit Engine

**6 Categories:**
1. **Performance** (Lighthouse) — LCP, FID/TBT, CLS, FCP, Speed Index
2. **SEO** (Lighthouse) — Meta tags, headings, structured data, crawlability
3. **Accessibility** (axe-core) — WCAG 2.1 AA, 57% auto-detection
4. **Security** (Custom) — HTTPS, security headers, SSL certificate, exposed files
5. **UX** (Custom) — Mobile responsiveness, tap targets, font sizes, viewport
6. **Best Practices** (Lighthouse) — Console errors, image sizes, HTTPS usage

**Scoring:**
- Weighted overall score: Performance 30%, SEO 20%, A11y 20%, Security 15%, UX 10%, Best Practices 5%
- Individual category scores: 0-100
- Color-coded: Good (90-100, green), Warning (50-89, amber), Poor (0-49, red)

### 3.2 AI Recommendations

Generated by GPT-4o for every failing issue:
- **What's wrong** — plain English explanation
- **Why it matters** — impact on users/SEO/performance
- **How to fix** — numbered steps (3-5)
- **Code examples** — HTML/CSS/JS snippets
- **Effort estimate** — S (under 1hr), M (1-4hrs), L (4+ hours)
- **Expected improvement** — score points (1-20)
- **Executive summary** — 2-3 sentence overview

### 3.3 PDF Reports

- Professional PDF generated via @react-pdf/renderer
- Sections: Overall Score, Category Breakdown, Issue Details, AI Recommendations
- Agency plan: White-label with custom logo and company name
- Downloadable from dashboard
- Async generation via BullMQ queue

### 3.4 Sharing

- Shareable public links (`sitegrade.io/r/{token}`)
- No login required to view
- Configurable expiry (default 30 days)
- View count tracking

## 5. Technical Architecture

### 5.1 Tech Stack

| Layer | Technology |
|-------|-----------|
| Frontend | Next.js 14 (App Router), Tailwind CSS, shadcn/ui, Framer Motion |
| Backend | Hono.js (Node.js), BullMQ v5 + Redis |
| Database | PostgreSQL (Supabase), Upstash Redis |
| ORM | Prisma v7 |
| Auth | Supabase Auth (JWT) |
| AI | OpenAI GPT-4o (primary), GPT-4o-mini (fallback) |
| Payments | Stripe |
| Email | Resend |
| Hosting | Vercel (web), Railway (api/workers) |
| Monitoring | Sentry, PostHog |

### 5.2 Architecture

Queue-driven, event-based architecture:

- **Vercel** — Next.js frontend (SSR + CSR + RSC)
- **Railway API** — Hono.js REST API (auth, CRUD, webhooks)
- **Railway Worker** — Background audit processing (Lighthouse, axe-core, custom checkers, AI generation)
- **BullMQ + Redis** — Job queue for audit processing and PDF generation
- **Supabase** — PostgreSQL, Auth, Storage

### 5.3 Audit Pipeline

1. User submits URL → API validates (SSRF check) → Creates audit record → Enqueues BullMQ job
2. Worker dequeues → Launches headless Chrome → Runs Lighthouse + axe-core + custom checkers
3. Worker calculates scores → Generates AI recommendations (cache-miss only) → Persists results
4. Client polls → Receives results → Renders score rings, issues, AI panel

## 8. Risk Assessment

| Risk | Severity | Likelihood | Mitigation |
|------|----------|-----------|------------|
| SSRF vulnerability | Critical | Medium | Multi-layer URL validation, blocked IP ranges |
| AI cost overrun | High | Medium | Aggressive caching (65%+ hit rate), GPT-4o-mini fallback |
| Competitor entry | Medium | High | Speed-to-market, AI differentiator, white-label moat |
| Supabase downtime | High | Low | Pro plan with 99.9% SLA |
| Lighthouse flakiness | Medium | Medium | Retry logic, median-of-3 runs (Phase 4) |
| Stripe integration bugs | High | Low | Extensive webhook testing, reconciliation job |
